Create Account Flow

This section describes the flow when a user chooses to create a new account to access personalized features within the application.

1. Entry Point

From the welcome screen, the user selects the Create an Account option to register a new user profile.

2. External Account Creation Portal

After selecting Create an Account:

  • The user is redirected to the official authentication portal: identity.unitedrugby.com in account creation mode.

  • The user can create an account using supported identity providers:

    • Email

    • Google

    • Apple ID

    • Twitter

3. Account Creation Options

The user can create an account using any of the supported identity providers (such as email, Google, Apple ID, or Twitter).

The user is then prompted to configure initial preferences, such as:

  • Favorite team selection

  • Subscription to newsletters

  • Receiving offers and announcements

4. Successful Account Creation

If account creation is successful:

  • The app detects that the user is authenticated.

  • A secure session token is stored locally for future authenticated requests.

  • User preferences (such as favorite teams) are fetched to personalize content.

  • The user is navigated to the main application interface.

5. Failed or Cancelled Account Creation

If account creation fails or the user cancels the process:

  • The user remains on the welcome screen.

  • No session is created, and the app continues in an unauthenticated state.

  • The user can retry account creation or choose to continue as a guest.

6. Summary Flow

Welcome Screen → Create an Account → Account Creation Portal → (Email / Social Sign-Up) → Preferences Selection → (Success → Main Application Interface) / (Failure → Welcome Screen)
