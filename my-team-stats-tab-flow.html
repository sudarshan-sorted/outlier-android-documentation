Stats Tab Flow

This section describes the user flow for the My Team → Stats tab, which shows aggregated team performance statistics.

1. Entry Point

  • User navigates to My Team section.

  • User selects the Stats tab from the top tab selector (e.g., Overview, Players, Stats).

2. Stats Tab Default State

When the Stats tab is opened:

  • Attack category is expanded by default.

  • Other stat categories are collapsed by default.

  • Team’s overall ranking/position (e.g., 15TH) is displayed in the expanded section.

3. Stats Categories Structure

Team statistics are grouped into collapsible categories:

  • Attack

  • Defence

  • Kicking

  • Discipline

  • Lineouts

  • Scrums

Each category:

  • Has a category title header.

  • Shows a chevron icon to expand or collapse the section.

4. Attack Category Flow (Expanded by Default)

When Attack is expanded, the following team metrics are displayed:

  • Points Scored

  • Tries Scored

  • Offloads

  • Metres Gained

  • Clean Breaks

  • Defenders Beaten

Each metric:

  • Displays a metric label on the left.

  • Displays a numeric value badge on the right.

5. Category Expand/Collapse Interaction

  • User taps on a category header (e.g., Defence, Kicking, Discipline).

  • The selected category expands to reveal metrics.

  • Other categories remain collapsed or expanded depending on accordion behavior.

6. Season Context

  • Team stats are season-based.

  • If a season selector exists in My Team context:

    • Changing the season updates all category data.

    • All metric values and rankings refresh dynamically.

7. Scroll Behavior

  • User scrolls vertically to view:

    • Metrics within expanded categories.

    • Additional stat categories below the visible viewport.

8. Exit Flow

  • User switches to Overview or Players tab.

  • User navigates back to Home or previous screen.
