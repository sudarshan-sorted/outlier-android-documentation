Login Flow

This section describes the flow when a user chooses to sign in and authenticate with the application.

1. Entry Point

From the welcome screen, the user selects the Sign In option to access personalized features.

2. External Authentication Portal

After selecting Sign In:

  • The user is redirected to the official authentication portal: identity.unitedrugby.com.

  • The portal opens in login mode, allowing the user to authenticate using supported identity providers:

    • Email

    • Google

    • Apple ID

    • Twitter

3. Successful Authentication

If the authentication is successful:

  1. The user is redirected back to the application.

  2. The app detects that the user is authenticated.

  3. A secure session token is stored locally for subsequent authenticated requests.

  4. The user is navigated to the main application interface.

This enables access to personalized content such as favorite teams and tailored feeds.

4. Failed or Cancelled Authentication

If the authentication fails or the user cancels the process:

  • The user remains on the welcome screen.

  • No session is created, and the app continues to operate in an unauthenticated state.

  • The user can retry the login process or choose to continue as a guest.

5. Summary Flow

Welcome Screen → Sign In → Authentication Portal → (Success → Main Application Interface) / (Failure → Welcome Screen)
