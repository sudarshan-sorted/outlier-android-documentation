Team Screen Flow

This section explains how users explore team statistics through collapsible stat categories and expandable team rankings with detailed performance metrics.

1. Default State

  • When the Team stats screen is opened:

    • Attack category is expanded by default.

    • First team in the list is expanded by default.

2. Stats Categories

Team statistics are grouped into the following collapsible categories:

  • Attack

  • Defence

  • Kicking

  • Discipline

  • Lineouts

  • Scrums

Each category is represented as a collapsible section with:

  • Category title

  • Expand / collapse chevron icon

3. Category Interaction

  • Categories are collapsed by default except Attack.

  • Users can:

    • Expand any category to view team rankings and metrics.

    • Collapse expanded categories to hide content.

4. Team Ranking List

Within each expanded category:

  • Teams are displayed in a ranked vertical list.

  • Each team row contains:

    • Rank number

    • Team logo

    • Team name

    • Expand chevron icon

5. Team Row Expansion

  • The first team row is expanded by default in every category.

  • Users can expand any team row to reveal detailed metrics.

  • Expanding a team row displays metric values relevant to the selected category.

6. Metrics Display

When a team row is expanded, a list of category-specific metrics is shown.

Example metrics under Attack:

  • Points Scored

  • Tries Scored

  • Offloads

  • Metres Gained

  • Clean Breaks

  • Defenders Beaten

(Other categories display their respective metric sets.)

7. User Interaction Summary

  • Default open state:

    • Attack category expanded

    • First team expanded

  • User can:

    • Expand / collapse stat categories

    • Expand / collapse team rows within categories
