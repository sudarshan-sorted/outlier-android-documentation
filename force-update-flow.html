Force Update Flow

This document describes the mandatory app update flow, which occurs every time the application is opened if a newer version is defined in the backend (Firebase Remote Config).

1. App Launch

  • User opens the app.

  • Splash screen is displayed while core services and configurations are initialized.

2. Force Update Check

  • The app fetches the latest version information from the backend.

  • If a newer version is required:

    • The user is redirected to the Play Store to update the app.

    • Access to the main interface is blocked until the update is completed.

  • If the app is up to date, the flow continues normally.

3. Post-Check

  • After confirming the app is up to date:

    • Permission handling is initiated (notifications, location, etc.).

    • Authentication status is checked (login, create account, or continue as guest).

4. Summary Flow

Open App → Splash Screen → Force Update Check → (Update Required → Play Store) / (Up-to-Date → Proceed with Permissions & Authentication)
