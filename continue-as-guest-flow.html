Continue as Guest Flow

This section describes the flow when a user chooses to explore the application without creating an account or signing in.

1. Entry Point

From the welcome screen, the user selects the Continue as guest option.

2. Guest Session Initialization

After selecting Continue as Guest:

  • The user is taken directly to the main application interface.

  • No authentication session is created.

  • User-specific data and preferences are not available.

3. Guest Experience

In guest mode:

  • The user can browse matches, teams, highlights, and general content.

  • Personalized features (such as favorite teams and tailored content feeds) are not enabled.

  • The user can choose to sign in or create an account later from the profile or settings section to enable personalization.

4. Summary Flow

Welcome Screen → Continue as Guest → Main Application Interface (Non-Personalized Experience)
