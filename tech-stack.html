URC Android - Technical Documentation Help

Tech Stack

This page describes the major libraries, SDKs, and tools used in the United Rugby Championship (URC) Android app.

UI & Frontend Framework

Jetpack Compose

Jetpack Compose is used to build the entire user interface of the application. It enables a declarative approach to UI development, making it easier to manage complex screen states and dynamic content updates. All screens, components, and layouts are designed using this framework to ensure consistency and scalability across the app.

Programming Language

Kotlin

Kotlin is the primary programming language used for the application. It provides modern language features, safety mechanisms, and improved developer productivity, making it suitable for building scalable and maintainable Android applications.

Build System

Gradle

Gradle is used to manage the build process, dependencies, and different build variants. It automates compiling, packaging, and distribution tasks for the Android app.

Media & Highlights Integration

Blaze Android SDK (WSC Sports)

The Blaze Android SDK is integrated to display player and match highlight stories. These highlights provide users with engaging visual content such as key moments, player stories, and match recaps directly within the app.

Dependency Management

Dagger / Hilt

Dagger and Hilt are used for dependency injection across the application. This helps manage object creation and lifecycle, improves modularity, and makes the app easier to test and maintain.

Real-Time Data Updates

Firebase

Firebase is used to deliver real-time live match updates, including scores, match events, and live scrolling commentary. This enables users to receive up-to-date match information without manually refreshing the app.

Backend Data Communication

GraphQL with Apollo

GraphQL is used to fetch data from the backend efficiently. Apollo provides type-safe models, query management, and caching, which improves performance and reduces unnecessary network calls. This setup ensures consistent and structured data flow throughout the app.

Image Loading

Coil

Coil is used to load images asynchronously within the application. It handles image fetching, caching, and rendering efficiently, ensuring smooth scrolling and reduced memory usage across image-heavy screens.

Authentication & User Personalization

Oval (Auth0)

Auth0 is used for user authentication. The authentication flow works as follows:

  • Users are redirected to identity.unitedrugby.com to sign up or log in.

  • After successful authentication, the app receives a JWT token.

  • The token is stored locally and used for subsequent authenticated requests.

  • User-specific information (such as favorite teams) is fetched using the token to personalize content and recommendations within the app.

This approach ensures secure authentication and enables personalized user experiences across the application.

Summary

The URC Android app is built using modern Android development tools and frameworks, focusing on scalability, real-time data delivery, personalized user experiences, and rich media content integration. The architecture leverages declarative UI, structured backend communication, secure authentication, and real-time updates to deliver a robust and engaging sports application.
