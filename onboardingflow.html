Onboarding Flow

This section describes the user journey from the first app launch to reaching the main application interface. It outlines how the app initializes, checks authentication status, and guides users through login or guest access.

1. App Installation & First Launch

After installation, when the user opens the app for the first time:

  • The application displays a splash screen while core services and configurations are initialized.

  • Basic system services such as analytics, messaging, and remote configuration are prepared in the background.

  • Initial data required for the home experience is preloaded to improve perceived performance.

2. Permission Handling

During the initial launch, the app requests certain permissions to enable key features:

  • Location Permission Used to provide region-specific ads, content, and localized experiences.

  • Notifications Permission
    Allows the app to send live match alerts, score updates, and important announcements.

If permissions are denied, the app continues to function but with limited notification & location capabilities.

3. Authentication Status Check

After initialization, the app checks whether the user is already authenticated:

  • If the user has previously logged in and a valid session exists, the app directly navigates to the main interface.

  • If no valid session is found, the user is shown the onboarding and authentication entry screens.

4. Welcome & Entry Screen

Unauthenticated users are presented with a welcome screen that introduces the app and provides two primary options:

  • Sign Up / Log In

  • Continue as Guest

This allows users to either personalize their experience or explore the app without creating an account.

5. Sign Up / Login Flow

When a user chooses to sign up or log in:

  1. The user is redirected to the official authentication portal (identity.unitedrugby.com).

  2. The user completes the authentication process using supported identity methods.

  3. After successful authentication, the app receives a secure token that represents the user session.

  4. The app retrieves user-specific preferences (such as favorite teams) to personalize content across the application.

This flow ensures secure authentication and enables tailored content recommendations.

6. Guest User Flow

If the user chooses to continue as a guest:

  • The user is taken directly to the main interface without authentication.

  • Personalized features (such as favorite teams and tailored content) are limited or unavailable.

  • The user can choose to log in later from the profile or settings section.

7. Main Application Entry

After authentication or guest selection, the user is navigated to the main application interface, where they can:

  • Browse matches and teams

  • View highlights and stories

  • Access live scores and match details

  • Manage profile and preferences

8. Summary Flow

Install App → Splash Screen → Permission Handling → Authentication Check → Welcome Screen → (Login / Guest) → Main Application Interface

Notes & Considerations

  • Authentication enables content personalization, such as favorite teams and tailored feeds.

  • Guest access provides a frictionless entry point but with limited personalization.
